OVERVIEW

This CMS is imagined as a series of blocks that make a blog. Every blog post has a header, subheader and some paragraphs. So does this CMS. There are 6 types of blocks: header, subheader, paragraph, multimedia block, code block and a quote block.

Every block appears in the final blog post differently. You can see that right away by previewing the blog in the upper right corner of this CMS. You can also see how they will look in the help section of each block. You can also rearrange the blocks however you like.

Be advised that every blog must have a title. Otherwise, it cannot be published. Although, short description and a cover image are optional, it is a good idea to include them since they make the blog post more descriptive for the users. Also, blog posts without a cover image will not be shown on the landing page so keep that in mind when writing your blog.

Have fun with it and I hope you like it.

TITLE, DESCRIPTION AND COVER IMAGE

A title is, as the name suggests, the title of the blog. It is required in order to publish a blog. A short description gives users a glimps into what the blog post is about and a cover image makes it stand out of all the other blogs.

For now, only unsplash images are allowed as cover images. Just copy/paste the shareable link into the cover text field and the image will appear automatically.

Be advised that only blogs with cover images will be displayed on the landing page. You can view how the blog looks and does your cover image fit into your blog in the preview.

MAIN HEADER BLOCK

Main header blocks are pretty straightforward. They signify a start to a section of your blog. To add a main header block, simply click on the menu on the left and a new main header block will be created. To delete it, hover over the header and click on the garbage icon. All the text is autosaved.

SUBHEADER BLOCK

Subheaders are the same as main header blocks. They are useful when you want to create a subsection of the section that your main header point to. All the text is autosaved.

TEXT BLOCK

You will likely use the text block most often. As the name suggests, you use it to put the actual text of your blog.

If you wish to change the appearance of your text, highlight the text and a floating menu will appear. With it, you can make some parts of the text bold, italic, make a list and other things.

If a paragraph block becomes too big, you can make the text invisible so you can work more easily on the other parts of the blog.

If your blog has a lot of text, images or code blocks and you haven't worked on it for a while, you can assign an internal name and a comment to a paragraph block. That way, when you return to work on it, you can easily see what was your intention.

CODE BLOCK

A code block allows you to create the code that your readers can execute. Every code block must have either an environment (Javascript, Go or some other) or an imported code project. An environment allows readers to execute a single piece of runnable code while a code project is a project that consist of many files that the user can then import. If you which to import a code project, you have to create it first in the code projects dashboard. Lets know explore more how can we create a new environment.
